# Phone-to-PC Flow

This is the practical bridge for your phone.

## What works now

- You can trigger a GitHub Actions workflow from your phone.
- The workflow records the action you requested.

## What still needs a computer

- If you want the action to run on your desktop, the desktop must be online.
- A self-hosted runner or local bridge is required for real desktop execution.

## Fastest setup path

1. Commit and push this repo to GitHub.
2. Open the repository on your phone.
3. Run the `Phone Trigger` workflow.
4. Later, connect the workflow to a self-hosted runner if you want desktop execution.

## Suggested next step

- Add a self-hosted runner on the desktop when you want the phone trigger to control local software.
